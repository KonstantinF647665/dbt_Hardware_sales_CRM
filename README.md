Проект представляет собой полноценный аналитический пайплайн для обработки данных CRM-системы (продажи оборудования). Основная цель — трансформация разрозненных сырых данных (CSV/Seeds) в чистые витрины данных, готовые для бизнес-аналитики в Power BI.

Результат: выявлен топ наиболее талантливых агентов, определены наиболее успешные циклы продаж, обнаружены просадки в продажах и агенты, занижающие стоимость (средний и общий показатель) ради кол-ва продаж

Основные KPI:

Total Sales Value (Net/Gross)

Win Rate по агентам и филиалам.

Средний цикл сделки (Days per Deal).

Анализ отклонения от РРЦ (Margin/Discount analysis).

Задачи:

Определить лидеров среди отделений / агентов / выявить узкие места (bottleneck), определить лидеров и отстающих

Технологии:
Key Stack:

Database: PostgreSQL 16+

Transformation: dbt (data build tool) Core

Language: SQL (Jinja)

Version Control: Git/GitHub

Архитектура данных
Проект реализован по принципу многослойной архитектуры DWH:

*Raw Layer (analytics_raw): Сырые данные, загруженные через dbt seed. Неизменяемый слой.

*Staging Layer (stg_): Первичная очистка, приведение типов, фильтрация "шума" и базовая бизнес-логика.

*Intermediate Layer (int_): Сложные джоины и промежуточные агрегации (например, объединение сделок с данными менеджеров).

*Mart Layer (fct_): Финальные витрины, оптимизированные для высокопроизводительных запросов из Power BI.

Ключевые фичи и оптимизация
В проекте реализованы продвинутые практики разработки:

Макросы (Jinja): Создан кастомный макрос convert_to_net для автоматизации расчета чистой выручки (очистка от налогов/VAT). Это обеспечивает принцип DRY (Don't Repeat Yourself).

Data Quality Testing:

Generic Tests: Уникальность (unique) и отсутствие пропусков (not_null) на ключевых ID.

Singular Tests: Кастомные SQL-тесты на бизнес-логику (проверка корректности дат закрытия сделок и валидация финансовых сумм).

Оптимизация производительности: Использование CTE-рефакторинга ("One Pass") для минимизации количества сканирований таблиц (Table Scans).


Как запустить проект
Клонировать репозиторий:

Bash
git clone https://github.com/your-username/crm-analysis.git
Настроить profiles.yml:
Убедитесь, что подключение к PostgreSQL настроено правильно (host, user, pass, dbname: crm_db).

Установить зависимости и собрать проект:

Bash
dbt deps
dbt seed
dbt build
Команда dbt build выполнит запуск моделей и прогон всех тестов одновременно.
